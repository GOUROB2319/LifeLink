# Contributing to LifeLink

Thank you for considering contributing to LifeLink! This document provides guidelines for contributing to the project.

### Reporting Bugs
- Use GitHub Issues
- Include detailed description
- Provide steps to reproduce
- Include screenshots if applicable
- Mention browser/OS version

### Suggesting Features
- Check existing issues first
- Explain the use case
- Describe expected behavior
- Consider implementation complexity

### Code Contributions
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test thoroughly
5. Submit a pull request

## ğŸ“ Code Guidelines

### JavaScript
- Use ES6+ syntax
- Use `const` and `let`, avoid `var`
- Add JSDoc comments for functions
- Keep functions small and focused
- Handle errors properly

### HTML
- Use semantic HTML5 elements
- Include ARIA labels for accessibility
- Use data attributes for i18n
- Keep markup clean and readable

### CSS
- Use TailwindCSS utility classes
- Follow existing design patterns
- Ensure responsive design
- Test dark mode

### Commit Messages
```
feat: Add notification system
fix: Resolve login validation bug
docs: Update setup instructions
style: Format code with prettier
refactor: Simplify error handling
test: Add validation tests
```
